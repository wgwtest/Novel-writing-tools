# 全局时间轴与章节工作台贯通设计 v1.0

## 1. 本文目的

本设计用于打通“全局时间画布”和“章节工作台”的联系。

此前原型已经分别验证了：

1. 全局时间画布可以表达真实时间、章节、剧情线、事件条和人物线。
2. 章节工作台可以表达章前输入、中段处理、章后输出。
3. Inspector 可以作为确认入口，避免点击对象后误跳转。

但两者之间尚未形成可走通路径。用户从全局时间画布选中一个章节后，应该能明确进入该章节的工作台，并看到该章节对应的输入事件、延续事件、章内处理和输出事件。

## 2. 核心原则

本轮不新增复杂页面，只打通对象路径：

```text
全局时间画布选中章节 -> Inspector 确认 -> 进入章节工作台
```

关键点：

1. 点击章节只选中，不自动跳转。
2. 右侧 Inspector 显示章节对象摘要。
3. 只有点击“进入章节工作台”，才进入对应章节工作台。
4. 章节工作台读取同一个章节 ID。
5. 工作台显示的是该章节的事件投影，而不是重新写一份孤立演示数据。

## 3. 对象关系

本轮涉及三个对象层：

1. 章节对象
   - 在全局时间画布中作为章节列显示。
   - 在章节工作台中作为当前工作对象。

2. 章节事件投影
   - 在章节工作台中显示章前输入、延续事件、章内新建事件、章后输出。

3. Inspector
   - 在全局时间画布中确认当前选中章节。
   - 在章节工作台中确认当前选中事件投影。

## 4. 路由设计

原型采用 hash 路由：

| 路由 | 含义 |
| --- | --- |
| `#timeline` | 全局时间画布。 |
| `#timeline / selected=C15` | 逻辑状态：全局时间画布中选中 C15。 |
| `#chapter/C15` | C15 章节工作台。 |

正式实现时可以替换为真实前端路由，但对象语义不应改变。

## 5. 全局时间画布行为

全局时间画布承担章节入口职责。

交互规则：

1. 点击章节头，选中章节列。
2. 被选中章节列高亮。
3. 右侧 Inspector 显示章节 ID、章节标题、输入、输出、事件投影摘要。
4. `进入章节工作台` 按钮变为可用。
5. 点击按钮后进入 `#chapter/<chapterId>`。

非目标：

- 不在全局时间画布中展开完整章节工作台。
- 不把章节点击变成自动跳转。
- 不在全局画布里直接编辑章节输入输出。

## 6. 章节工作台行为

章节工作台根据章节 ID 读取数据。

工作台必须显示：

1. 当前章节 ID 与标题。
2. 当前章节真实时间范围。
3. 章前输入。
4. 中段处理。
5. 章后输出。
6. 右侧 Inspector。
7. 返回全局时间轴按钮。

工作台中的事件条仍然遵守前序规则：

- 已结束历史事件只作为来源。
- 延续事件是一条事件投影，不拆成多段。
- 章内新建事件由输入事件或延续事件驱动。
- 输出事件进入后续章节或全局事件流。

## 7. 数据读取规则

本轮原型采用同一份前端静态数据模拟系统记录：

1. `chapters`
   - 提供章节 ID、标题、真实时间、输入、输出。

2. `timelineEvents`
   - 提供全局时间画布里的事件条。

3. `workbenches`
   - 提供章节工作台中的章节事件投影。

正式系统中，应由以下对象记录生成这些视图：

1. `chapter`
2. `event`
3. `chapterEventProjection`
4. `eventRelation`
5. `storylineEventMembership`

## 8. 原型验收点

本轮原型合格条件：

1. 打开页面默认进入全局时间画布。
2. 点击 C15 后不会跳转，只会选中 C15。
3. 右侧 Inspector 显示 C15 的章节对象信息。
4. 点击“进入章节工作台”后进入 C15 工作台。
5. C15 工作台标题、时间、输入事件、延续事件、输出事件与 C15 对应。
6. 工作台中可以点击事件投影，右侧 Inspector 显示事件投影信息。
7. 可以从工作台返回全局时间画布。

## 9. 非目标

本轮不处理：

1. 完整事件详情页。
2. 事件对象持久化保存。
3. 章节工作台内复杂拖拽编辑。
4. 多章节之间的批量切换。
5. 后端数据结构实现。

本轮只打通入口路径和对象 ID 贯通。
