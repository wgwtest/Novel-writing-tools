# 对象可视化要素与颜色编码规则 v1.0

## 1. 本文目的

本文用于固定全局时间画布中的对象可视化规则。

此前原型已经能显示章节、剧情线、事件块和人物子线，但还缺少一条稳定规则：每一种视觉要素到底表达哪个系统对象的哪个属性。如果颜色、线型、长度、虚实、人物标记都临时混用，画布会看起来像随机配色，后续真实数据接入也会不断产生解释成本。

本文基于 `2026-06-23-叙事验证工具-对象可视化要素示意-v1` 的评审结果，确认以下方向：

```text
故事线采用 B 方案：故事线身份色优先，事件状态用局部标记表达。
事件、人物、人物片段按示意图中的推荐规则执行。
```

## 2. 总体编码原则

全局时间画布的视觉编码按以下优先级执行：

| 优先级 | 视觉要素 | 表达对象 | 规则 |
| --- | --- | --- | --- |
| 1 | 横向位置 | 时间 | 只表达真实时间或章节覆盖位置。 |
| 2 | 纵向位置 | 故事线 / 人物片段归属 | 主行是故事线，展开后子行是人物片段。 |
| 3 | 长度 | 事件或人物片段持续范围 | 由 `startAt / endAt` 决定。 |
| 4 | 主色 | 故事线身份 | 由故事线 `baseColor` 决定，不被事件状态覆盖。 |
| 5 | 线型 | 正文可见性 | 实线表示正文直接呈现，虚线表示后台存在或未直接写出。 |
| 6 | 局部状态标记 | 风险、警示、关键、延续等状态 | 使用边条、角标、底边、图标或小标签表达。 |
| 7 | Inspector | 详细属性 | 摘要、人物、地点、组织、输入输出、风险等放右侧面板。 |

这条优先级用于避免同一块颜色同时表达故事线、事件状态、人物身份和可见性。

## 3. 章节对象

章节是正文切片和章节函数入口，不是故事线，也不是事件。

### 3.1 系统字段

章节对象建议记录：

```yaml
chapter:
  id: C14
  title: 会后吸收
  startAt: 2025-07-15T12:00
  endAt: 2025-07-17T18:00
  inputEventIds: []
  continuingEventIds: []
  createdEventIds: []
  outputEventIds: []
```

### 3.2 可视化要素

| 要素 | 用途 |
| --- | --- |
| 章节覆盖区 | 表达该章在真实时间轴上的正文切片。 |
| 章节标题 | 显示 C 编号与章节名。 |
| 选中背景 | 点击章节后突出整段章节区域。 |
| Inspector 入口 | 选中后显示“进入章节函数”。 |

章节不使用颜色表达事件状态，也不承担人物线颜色。

## 4. 故事线对象

故事线是全局时间轴中的一级叙事轨道。用户本轮确认采用 B 方案：

```text
故事线身份色优先。
事件状态只做局部标记。
```

### 4.1 系统字段

故事线对象建议记录：

```yaml
storyline:
  id: interview
  name: 采访线
  role: main
  summary: 围绕皮革厂与病患群体展开的采访推进。
  baseColor: teal
  priority: 1
  expanded: true
  characterIds:
    - yeyiren
    - zhanghangyuan
```

### 4.2 可视化要素

| 要素 | 用途 |
| --- | --- |
| 轨道行 | 表达事件所属故事线。 |
| 轨道名 | 显示故事线名称。 |
| `baseColor` | 作为该故事线下事件块的主色。 |
| 轨道背景 | 弱提示当前行范围。 |
| 展开状态 | 控制人物子线是否出现。 |
| 人物子线容器 | 展开后承载人物片段。 |

### 4.3 颜色规则

故事线颜色必须稳定。

同一条故事线中的普通事件、警示事件、风险事件，主填充色仍然来自该故事线。事件状态只能通过以下附加层表达：

风险红和警示橙属于状态保留色，不应直接作为故事线身份色使用。否则用户会无法判断某个红色块到底表示“张航远线”，还是表示“风险事件”。

| 状态 | 推荐表达 |
| --- | --- |
| 普通 | 故事线主色填充 + 主色边框。 |
| 警示 | 保持故事线主色，增加琥珀色左边条或 `WARN` 角标。 |
| 风险 | 保持故事线主色，增加红色底边、角标或外边框。 |
| 关键 | 保持故事线主色，增加粗边框或关键节点标记。 |
| 后台 | 保持归属，但使用虚线边框和弱填充。 |

不采用“事件 tone 直接决定整块颜色”的规则。

## 5. 事件对象

事件是叙事推进的最小可追踪对象。事件块是事件在全局时间画布中的投影，不等于事件全部内容。

### 5.1 系统字段

事件对象建议记录：

```yaml
event:
  id: E021
  title: 会议结果落地
  storylineId: interview
  startAt: 2025-07-13T08:00
  endAt: 2025-07-15T12:00
  chapterIds:
    - C13
    - C14
  type: story
  lifecycleStatus: active
  visibility: explicit
  tone: warn
  participantIds: []
  locationIds: []
  organizationIds: []
  causeEventIds: []
  effectEventIds: []
  summary: ''
  input: ''
  process: ''
  output: ''
  risk: ''
```

### 5.2 画布块内显示

事件块内只显示高频识别信息：

| 要素 | 用途 |
| --- | --- |
| 标题 | 事件短标题。 |
| 事件 ID / code | 便于和文档、Inspector 对齐。 |
| 章节覆盖 | 显示 C 范围或章节编号。 |
| 少量状态标记 | 例如 `WARN`、`RISK`、延续边条。 |
| 关系端点 | 用于连接前置或后置事件。 |

### 5.3 Inspector 显示

选中事件后，Inspector 显示：

1. 事件 ID
2. 标题
3. 所属故事线
4. 真实时间范围
5. 章节范围
6. 参与人物
7. 地点
8. 组织
9. 摘要
10. 输入
11. 输出
12. 风险
13. 进入事件详情按钮

### 5.4 详情页显示

事件详情页显示完整内容：

1. 起因
2. 过程
3. 结果
4. 前置事件
5. 后置事件
6. 关系编辑
7. 事件正文记录或编辑区

## 6. 人物对象

人物是系统字典对象，不是时间条。人物对象本身没有必然的起止时间。

### 6.1 系统字段

```yaml
character:
  id: zhanghangyuan
  name: 张航远
  shortName: 张
  role: 年轻记者
  identityColor: red
  relatedStorylineIds: []
  relatedEventIds: []
```

### 6.2 可视化要素

| 要素 | 用途 |
| --- | --- |
| 人物色点 | 在人物标签、Inspector、参与者列表中辅助识别。 |
| 简称 | 在空间紧张时显示。 |
| 参与者标签 | 显示事件涉及哪些人物。 |
| 筛选状态 | 后续可用于只看某人物相关事件。 |

人物色不决定事件块主色。事件块主色仍由故事线决定。

## 7. 人物片段对象

人物片段是“某个人在某条故事线中的一段参与或后台状态”。它是时间对象，依附于故事线展开层。

### 7.1 系统字段

```yaml
characterSegment:
  id: seg_interview_zhanghangyuan_001
  storylineId: interview
  characterId: zhanghangyuan
  eventId: E012
  startAt: 2025-07-08T08:00
  endAt: 2025-07-10T02:00
  visibility: explicit
  actionSummary: 补充追访和核实名单。
  stateChange: 对皮革厂叙事的怀疑上升。
```

### 7.2 可视化要素

| 要素 | 用途 |
| --- | --- |
| 低高度子行 | 明确低于主故事线层级。 |
| 人物色边线 | 表达人物身份。 |
| 实线框 | 正文写到的人物行为。 |
| 虚线框 | 后台存在但正文未直接写出的状态。 |
| 关联事件 | 说明该人物片段关联哪个事件。 |

人物片段可以比某个单独事件长，但它不扩大故事事件边界。它表达人物状态或行为持续，不表示正文事件仍然在发生。

## 8. 当前确认结论

本轮确认：

1. 故事线采用 B 方案。
2. 故事线主色是一级视觉编码。
3. 事件状态使用局部标记，不覆盖故事线主色。
4. 人物对象和人物片段对象分开。
5. 人物色只作为人物识别辅助。
6. 人物片段使用低高度、人物色边线、虚实线型表达。

## 9. 后续实现影响

后续需要同步修改：

1. `schema.json`
   - 增加或明确 `storyline.baseColor`。
   - 区分 `event.tone` 和 `event.visualState`。
   - 区分 `character.identityColor` 和 `characterSegment`。

2. `story.json`
   - 故事线记录主色。
   - 事件只记录状态，不再靠状态决定主色。
   - 人物片段应从普通 `event.kind=character` 中独立出来，或至少在语义上明确为 `characterSegment`。

3. `index.html`
   - 渲染事件块时先读取所属故事线主色。
   - 再按事件状态叠加边条、角标、虚线或底边。
   - 人物片段只在展开层使用人物色。

## 10. 非目标

本文不决定：

1. 最终色值。
2. 是否立即重构所有 C1-C5 真实数据。
3. 是否增加组织、地点、证据的独立画布层。
4. 事件详情页具体编辑表单。

本文只固定对象可视化编码规则。
