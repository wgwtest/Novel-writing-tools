# 章节事件加工台与对象编辑边界设计

版本：v0.3  
状态：供原型评审  
适用范围：`验证工具` 内的叙事验证工具原型与后续验证性小程序  
关联原型方向：v14 章节事件加工台原型  

## 1. 本次修订目的

前序原型验证了三类能力：

1. 全局时间线：以真实时间、章节、故事线和人物线为基础，做宏观定位。
2. 中心事件知识图谱：以一个事件为中心，左侧显示来源事件，右侧显示输出事件。
3. 事件对象编辑：以事件为对象，检查触发条件、人物动作、冲突机制、动态变化和输出风险。

本次修订新增第四类能力：章节为中心的事件加工台。

同时固化一条边界规则：

```text
全局时间线不再叠加事件关系箭头。
```

全局时间线只负责定位，不负责表达事件因果。事件关系、事件内部结构和章节内部加工逻辑都应进入独立页面。

## 2. 四类界面边界

| 界面 | 主对象 | 主要问题 | 不承担什么 |
| --- | --- | --- | --- |
| 全局时间线 | 故事全局进度 | 事件、章节、人物线在时间和章节中的位置 | 不画事件因果箭头，不做内部编辑 |
| 中心事件知识图谱 | 单个事件及其一阶关系 | 这个事件从哪里来、输出到哪里去 | 不编辑章节内部加工 |
| 事件对象编辑 | 单个事件 | 当前事件如何被触发、如何发生、如何输出 | 不处理整章多事件组合 |
| 章节事件加工台 | 单个章节 | 本章如何接收并列事件输入、内部组合、输出事件集 | 不把输入事件强行画成因果链 |

## 3. 章节为中心的定义

章节是正文函数。章节接收外部事件输入，在章节内部完成组合与处理，然后输出外部事件。

推荐表达：

```text
输出事件集 = 章节函数(并列输入事件集)
```

与事件对象编辑不同，章节编辑器的左侧输入不是因果链，而是输入池。

```text
并列输入事件 A
并列输入事件 B
并列输入事件 C
        ↓
章节内部处理逻辑
        ↓
输出事件 X
输出事件 Y
输出事件 Z
```

## 4. 并列输入规则

章节左侧输入区只表达“哪些外部事件进入本章”，不表达这些输入事件之间的因果关系。

输入事件可以来自不同故事线：

- 采访线事件
- 大仙祠线事件
- 电视台线事件
- 张航远线事件
- 职业病证据线事件

这些事件进入章节时是并列材料。它们之间是否存在因果关系，应在事件知识图谱中查看，而不是在章节输入区强行连接。

## 5. 内部组合规则

章节内部才有逻辑。章节加工台需要表达本章如何处理输入事件。

常见内部操作包括：

| 操作 | 含义 |
| --- | --- |
| 承接 | 把外部事件带入本章，但不改变其根本状态。 |
| 合流 | 多个并列输入在本章中被放进同一场景或同一冲突。 |
| 冲突化 | 让并列输入产生对抗、压力或选择。 |
| 转译 | 把一个事件的事实转换成另一条线可使用的口径。 |
| 显形 | 让此前隐性状态在正文中变得可见。 |
| 悬置 | 有意识地不终结某条输入，只保留后续入口。 |
| 终结 | 在本章内关闭某个输入事件。 |
| 生成 | 输出新的事件，作为后续章节的输入。 |

内部组合可以用蓝图式节点表达，但它不是程序流程图。节点语义服务于小说章节分析。

## 6. 外部输出规则

章节右侧输出区表达“本章产出了哪些外部事件”。

输出事件可以是：

- 新建事件
- 被补充后的原有事件
- 被重新包装后的事件
- 被终结的事件
- 被悬置但保留入口的事件

输出区也不必强行画出输出事件之间的因果关系。输出事件之间的关系仍交给中心事件知识图谱处理。

## 7. 事件编辑与章节编辑的差异

### 7.1 事件对象编辑

事件对象编辑关注一个事件内部是否成立：

```text
来源事件 -> 触发条件 -> 人物动作 -> 冲突机制 -> 动态变化 -> 输出事件
```

事件页可以画成较明确的处理链，因为事件本身就是一个独立过程。

### 7.2 章节事件加工台

章节事件加工台关注一章如何处理多个事件：

```text
并列输入池 -> 章节内部组合 -> 输出事件池
```

章节页不应把输入事件连成链。章节内部处理区可以有逻辑线，但输入池和输出池应保持集合感。

## 8. 推荐页面结构

章节事件加工台采用三段式：

```text
左侧：输入事件池
中部：章节内部加工台
右侧：输出事件池
底部：人物 / 地理 / 组织上下文
```

中部可以进一步分为：

- 场景承接
- 事件合流
- 冲突化处理
- 口径转译
- 风险悬置
- 输出生成

## 9. 验收问题

v14 原型需要评审：

1. 全局时间线不再画关系箭头后，边界是否更清楚。
2. 章节输入池是否能表达“并列输入”，而不是误读成因果链。
3. 中部章节加工台是否能表达“内部组合带逻辑”。
4. 输出池是否能表达章节产物，而不是把输出事件互相强连。
5. 章节加工台与事件对象编辑是否形成清晰区别。
