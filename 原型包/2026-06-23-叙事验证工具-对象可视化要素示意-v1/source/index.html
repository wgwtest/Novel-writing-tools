ENCODING ORDER

视觉编码优先级

越靠上越稳定，越不应该被临时状态覆盖。

1
位置横向是真实时间，纵向是所属故事线。
2
长度事件或人物片段持续范围。
3
归属色故事线身份色优先，人物色只用于子线。
4
线型实线表示正文可见，虚线表示后台存在。
5
状态标记风险、警示、延续、完结使用角标或边条。
6
Inspector详细属性放右侧，不全部塞进块内。
故事线身份色：稳定表达归属。
虚线：后台状态或未在正文直接呈现。
风险色：只做局部强调，不替换对象身份。

STORYLINE COLOR OPTIONS

故事线可视化的三种可选规则

这里故意用同一条“采访线”演示。重点是看颜色到底表达“故事线归属”还是“事件状态”。

A. 状态色优先

当前原型接近这种。事件按 main / warn / risk 直接换整块颜色。

采访线 身份被状态色稀释
采访启动E001 / C1
符纸异常E006 / C3
追访边缘化E012 / C4-C5
优点是状态醒目。问题是同一条线内出现多种主色，用户会误以为它们属于不同线。

B. 故事线色优先

建议方案。整块保持故事线身份色，状态用边条、角标、边框表达。

采访线 青绿色稳定表示归属
采访启动E001 / C1
符纸异常E006 / C3WARN
追访边缘化E012 / C4-C5RISK
归属最稳。风险、警示仍可见，但不会破坏故事线阅读。这最适合当前全局时间画布。

C. 中性色加标签

高密度方案。块体尽量中性，状态和归属都放到标签、边线或左侧轨道。

采访线 低噪声，但识别慢
采访启动E001 / C1
符纸异常E006 / C3WARN
追访边缘化E012 / C4-C5RISK
适合报告或打印，不适合复杂交互。因为扫视时不容易区分不同故事线。
故事线主色

每条故事线固定一个 baseColor。它决定本线事件块填充、轨道边条和选中态基色。

状态附加层

风险、警示、关键节点不换整块色，用左侧窄条、角标、底边或边框强调。

展开层

人物子线只能在故事线展开后出现。它使用人物色，但必须比主故事线更细、更轻。

选择建议

优先选 B。它最符合“故事线是一级对象，状态是二级信息”的系统层级。

EVENT BLOCK ANATOMY

一个事件块最多应该承载什么

事件块是全局画布上的最小叙事节点。它需要能被选中、被连接、被 Inspector 解释，但不能把全部字段塞进块内。

会议结果落地

E021-E028 / C13-C14 / 采访线
标题 只放短标题。摘要、起因、过程、结果放 Inspector。
时间与章节 横向长度来自 startAt/endAt；小字显示事件编号和章节覆盖。
状态边条 警示或风险使用局部边条，不替换故事线主色。
关系端点 左端接前置事件，右端接后置事件。连线可选中、可删除。
Inspector 事件 ID、人物、地点、组织、输入、输出、风险和进入详情按钮放在右侧。

CHARACTER OBJECT

人物对象与人物片段的区别

人物是字典对象，人物片段是时间对象。两者不能混成一条“大人物线”。

人物对象

记录姓名、简称、身份色、所属组织、角色说明。它没有必然的 startAt/endAt。

显示位置左侧人物标签、Inspector、筛选器
主视觉小色点、简称、参与者标签
不承担不决定故事事件主色

人物片段对象

记录人物在某条故事线中的一段参与。它必须有 trackId、characterId、startAt 和 endAt。

显示位置故事线展开后的子行
主视觉低高度条、人物色边线、虚实线型
不承担不扩大正文事件范围

选择点

需要确认人物片段是否只显示“有意义的动作段”，还是也允许显示“后台状态段”。如果允许后台段，虚线必须成为强规则。
