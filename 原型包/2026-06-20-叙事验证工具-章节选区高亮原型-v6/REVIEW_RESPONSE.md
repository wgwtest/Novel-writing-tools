# v5 评审意见响应

## 用户意见摘要

用户认可 v5 的方向，但指出画布中贯穿显示区的绿色竖线会造成错误理解：

1. 它像剪辑软件里的当前播放指针。
2. 当前工具并没有“当前时间点”或“写到哪里”的语义。
3. 选择章节后，应突出整个章节背景框，而不是出现一条线。
4. 该竖线视觉权重过高，会抢占事件和章节本身。

## 判断

意见成立。v5 将视频剪辑软件的“多轨时间线”借鉴到了小说结构工具中，但错误保留了播放头式视觉。小说管理工具当前关注的是章节单元、事件块和多轨对齐关系，而不是播放进度或当前时间点。

## v6 处理方式

- 删除默认 C15 贯穿竖线。
- 删除默认 C15 顶部播放头式标签。
- 删除 C15 在未选中状态下的特殊底部强调条。
- 新增 `drawSelectedChapterColumn`，只在章节被选中时绘制整列背景和边框。
- 事件选中仍然只强化事件块本身，不产生全局竖线。
- 文案从“当前画布状态”弱化为“画布状态”，并明确“指针不显示”。
- README 改为以“章节选区”作为本版评审核心。
