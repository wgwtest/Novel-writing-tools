# 叙事验证工具 - 事件树与事件检查工作台原型 v12

## 元信息

- 版本：v12
- 状态：供用户评审
- 继承版本：v11 事件章节三段式关系原型
- 目标画板：1920 x 1080
- 目标入口：`source/index.html`

## 本版定位

V12 回应两个问题：

1. 首页全局画布需要表达事件前后置关系，但不应把所有箭头直接叠到时间线上造成混乱。
2. 事件页不能只是文字说明，触发条件、人物动作、冲突机制、动态变化应成为可检查的结构。

## 三个视图

### 01-首页时间线局部连线-1920x1080.png

时间线仍是首页默认模式。只有选中事件周围显示输入与输出箭头，不全量显示所有关系线。

需要判断：局部连线是否能补足时间线的因果表达，同时不破坏原有多轨阅读。

### 02-首页事件树模式-1920x1080.png

事件树是首页的另一种展示形态，用来专门看前后置、分裂、合流。

需要判断：这个模式是否适合作为“时间线 / 事件树”的切换视图。

### 03-事件检查工作台-1920x1080.png

事件详情页中部改成可检查结构，包括：

- 触发条件
- 人物动作
- 冲突机制
- 动态变化

每一块都包含字段、证据、状态和问题提示。

需要判断：这种检查工作台是否比 V11 的段落说明更接近可编辑、可验证的事件界面。
