# v14 设计回应

## 用户意见

用户确认：

1. 全局时间线不要再修改。
2. 事件关系箭头会把全局时间线搞乱。
3. 当前可接受的方向包括中心事件知识图谱和事件对象编辑。
4. 还需要“章节为中心”的编辑。
5. 章节编辑的核心是接收外部事件输入，输入事件是并列的，没有逻辑关系。
6. 章节内部组合才带逻辑，再向外输出事件。

## 判断

该意见成立。章节对象和事件对象不能使用同一种因果链表达。

事件对象编辑可以表达为：

```text
来源事件 -> 当前事件处理 -> 输出事件
```

章节对象编辑应表达为：

```text
并列输入事件池 -> 章节内部组合处理 -> 输出事件池
```

因此章节输入池和输出池不应互相连线，逻辑只发生在章节内部处理区。
