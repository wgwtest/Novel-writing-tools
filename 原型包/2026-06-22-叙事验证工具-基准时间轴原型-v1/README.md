- 版本：基准时间轴原型 V1
- 状态：待用户确认

- 目标画板：1920 x 1080

## 本版定位

本版把 `v9` 从临时编号升级为“基准时间轴原型 V1”。它继续承担全局故事流入口：故事由事件组成，但事件必须落在真实时间底盘上；章节不是横轴本体，而是覆盖在真实时间上的正文切片。

本版不重画、不简化全局时间画布。此前已确认的缩放、拖拽、章节覆盖、故事线、人物子线、人物虚段和 Inspector 逻辑继续保留。

新增能力是：选中任意章节后，右侧 Inspector 的“进入章节函数”按钮会进入章节管理页面，用该章节 ID 和标题作为上下文。

## 运行方式

本版已经支持从 JSON 读取结构化数据。推荐从 `source/` 目录启动本地静态服务器：

```powershell
cd C:\OpenCodeWorkSpace\TestProject\文章重写\验证工具\原型包\2026-06-22-叙事验证工具-基准时间轴原型-v1\source
python -m http.server 4173
```

然后打开：

```text
http://127.0.0.1:4173/index.html
```

如果直接用 `file://` 打开，浏览器可能禁止 `fetch("./data/story.json")`。页面会自动退回内置演示数据，但这不适合后续真实数据构建。

本版当前 JSON 样板数据时间范围为：

```text
2025-07-05 09:00 -> 2025-07-10 02:00
```

这对应《致命采访》第 1 章到第 5 章的真实样板段：从皮革厂专题前期准备、采访当天厂门口丢符，到电视台正式安排跟进报道和林倩入组。该样板段用于验证结构化数据接入方式；后续 C6-C20 应继续扩展 `story.json`。

## 非目标

- 不做复杂日历编辑器。
- 不做拖拽修改日期。
- 不做自动保存。
- 不修订正式小说时间线。
- 不把演示日期写成最终设定。
- 不用简化版时间轴替代当前基准画布。

## 设计依据

- 用户要求：时间轴应能对应真实时间，并且起止时间可设置。
- 用户补充：第二卷可从 2025 年 7 月第二周附近开始，先选一个时间作为起止范围。
- v8 基准：事件线只显示正文可见事件；人物条区分实段和后台虚段。
- 设计判断：真实时间轴应成为底盘，章节覆盖层只是写作切片。

## 核心设计

### 1. 真实时间轴

画布顶部第一层显示真实日期刻度。默认从 `2025-07-07 08:00` 到 `2025-08-01 00:00`。

工具栏提供起止时间输入框，点击“应用”后重新计算画布宽度和所有对象位置。

日期刻度只作为顶部标尺，不贯穿主画布。

日期标尺与章节覆盖层是两个不同视觉层：日期标尺只承担真实时间提示，不参与主画布网格划分。

### 2. 章节覆盖层

章节不再等宽。每章根据 `startAt / endAt` 显示为真实时间轴上的覆盖区。

点击章节只选中章节覆盖区，不自动进入章节函数。

主画布的纵向网格以章节起点为准，而不是以每天为网格单位，也不同时绘制每章结束线。章节结束时间仍然保留在章节覆盖区宽度中；如果相邻章节之间存在真实时间空隙，该空隙只在章节覆盖层中体现，不额外生成一根贯穿主画布的竖线。

### 3. 事件线与人物条

事件、人物实段、人物虚段都使用真实时间字段：

```text
startAt
endAt
```

事件线继续只表示正文可见事件。人物条继续区分：

- 实框：正文写到的人物行为。
- 虚框：后台存在但未直接写出的人物状态。

虚框必须是完整虚线框，不再绘制实段使用的底部强调线。

### 4. Inspector

选中事件或人物条后，Inspector 显示：

- 真实时间范围。
- 覆盖章节。
- 所属故事线。
- 人物条的 `EXPLICIT / BACKGROUND` 状态。

选中章节后，Inspector 显示章节输入、章节内部运作和章节输出摘要；只有点击“进入章节函数”后，才进入章节管理页面。

### 5. 章节函数入口

章节函数入口从全局时间轴发起：

```text
全局时间轴 -> 选中章节 -> Inspector 确认 -> 进入章节函数
```

入口会把章节 ID、章节标题、起止时间传给 `chapter-workbench.html`。章节工作台负责处理本章的“章前输入 / 中段处理 / 章后输出”，但不反向简化全局时间轴。

### 6. 结构化数据入口

`source/data/story.json` 是当前原型的数据入口。它包含：

- `timelineConfig`：全局时间范围。
- `chapters`：章节覆盖层。
- `characters`：人物字典。
- `tracks`：故事线和人物子线配置。
- `events`：故事事件与人物参与片段。

`source/data/schema.json` 定义这些字段的基本结构。后续从第一章到第二十章抽取真实数据时，应优先更新 `story.json`，不要继续把新数据写死在 HTML 里。

当前 `story.json` 已接入 C1-C5 样板数据，来源包括：

- `第002卷-第001章.txt` 至 `第002卷-第005章.txt`
- `致命采访-分章大纲.txt`
- `致命采访-分章简纲.txt`
- `致命采访-事件脉络.md`

样板段对象数量：

- 章节：5
- 剧情线：7
- 事件与人物片段：35

## 原型到实现映射

- 页面归属：叙事验证工具 / 多轨时间轴工作台。
- 主对象：真实时间轴、章节覆盖、事件线、人物条。
- 数据来源：`source/data/story.json` 中的 C1-C5 真实样板数据；如果 JSON 读取失败，页面才退回内置演示数据。
- 验收方法：打开 `source/index.html`，检查默认视图、时间范围设置、选中态 Inspector；选中任意章节后点击“进入章节函数”，进入 `source/chapter-workbench.html`。

## 允许偏差

- 示例日期可调整。
- 章节覆盖时间可根据正式时间线重设。
- 日期刻度密度可根据缩放比例继续细化。
- 顶部日期标尺样式可调整，但不得变成主画布网格。

## 不可接受偏差

- 不能回退为等宽章节轴。
- 不能用章节编号直接决定事件块位置。
- 不能让日期刻度贯穿主画布并替代章节网格。
- 不能让人物后台虚段扩展故事线事件边界。
- 不能给人物虚段额外叠加实段式底部强调线。
- 不能把演示时间误写成正式设定。
