# v10 设计回应

## 用户意见

用户要求先把对象详情页与 Inspector 路由设计落到文档，再制作几个原型效果图。

核心要求包括：

1. 故事轴事件应包含基础摘要、参与人物、起因、过程、结果。
2. 事件进入后应以“事件”为对象形成事件级页面。
3. 章节函数应进入独立页面，明确展示输入、处理、输出。
4. 人物线展示方式暂未完全确定，应先保留设计分析。

## 判断

该意见成立。当前 v9 的右侧 Inspector 和同页面详情面板不足以承载对象级分析。继续往 Inspector 里堆字段会造成三个问题：

- 画布右侧面板过载。
- 事件、章节、人物线三类对象边界混在一起。
- “进入详情”没有真正形成页面级工作区。

## 本版处理

- 新增设计说明：`../../设计说明/2026-06-20-对象详情页与Inspector路由设计-v0.1.md`。
- 创建 v10 独立原型包，不覆盖 v9。
- 用 hash 路由模拟四个状态：
  - `#timeline`
  - `#event`
  - `#chapter`
  - `#lane`
- 生成四张 1920 x 1080 效果图供评审。

## 保留风险

- 人物线页仍是低承诺设计，后续需要用户继续确认。
- 本版不实现保存和真实数据结构。
- 本版视觉样式不代表最终 UI 标准，主要验证对象边界和页面逻辑。
